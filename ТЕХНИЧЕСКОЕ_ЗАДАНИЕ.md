# Техническое задание

## Проект: SleepMind - AI-система мониторинга сна и стресса

---

## 1. Общие сведения

### 1.1 Название проекта

**SleepMind** - интеллектуальное мобильное приложение для мониторинга и анализа качества сна и уровня стресса

### 1.2 Цель проекта

Разработка кроссплатформенного мобильного приложения с серверной частью для отслеживания качества сна, уровня стресса, ведения дневника здоровья и получения персонализированных рекомендаций на основе AI-анализа.

### 1.3 Основные технологии

| Компонент                      | Технология                                 |
| --------------------------------------- | ---------------------------------------------------- |
| Мобильное приложение | React Native (Expo) + TypeScript                     |
| Backend                                 | Java Spring Boot                                     |
| База данных                   | PostgreSQL                                           |
| Аутентификация            | JWT (JSON Web Tokens)                                |
| AI-сервис                         | Python (TensorFlow/PyTorch) - опционально |
| Контейнеризация          | Docker + Docker Compose                              |

## 2. Функциональные требования

### 2.1 Модуль аутентификации и авторизации

#### 2.1.1 Регистрация пользователя

- **API**: `POST /api/auth/register`
- **Входные данные**:
  - `name` (string, обязательно) - полное имя пользователя
  - `email` (string, обязательно) - email (уникальный)
  - `password` (string, обязательно) - пароль (минимум 6 символов)
- **Выходные данные**:
  - Объект пользователя (id, name, email, createdAt)
  - JWT токены (accessToken, refreshToken, expiresIn)
- **Валидация**:
  - Email должен быть уникальным
  - Пароль должен содержать минимум 6 символов
  - Email должен соответствовать формату

#### 2.1.2 Вход в систему

- **API**: `POST /api/auth/login`
- **Входные данные**:
  - `email` (string)
  - `password` (string)
- **Выходные данные**: JWT токены + данные пользователя
- **Ошибки**: 401 при неверных credentials

#### 2.1.3 Обновление токена

- **API**: `POST /api/auth/refresh`
- **Входные данные**: `refreshToken`
- **Выходные данные**: Новая пара токенов

#### 2.1.4 Выход из системы

- **API**: `POST /api/auth/logout`
- **Требуется**: Authorization Bearer token
- **Действие**: Инвалидация токенов

#### 2.1.5 Профиль пользователя

- **GET /api/auth/me** - получение профиля
- **PUT /api/auth/profile** - обновление профиля (name, avatar)
- **POST /api/auth/change-password** - смена пароля
- **POST /api/auth/forgot-password** - запрос на сброс пароля

---

### 2.2 Модуль дневника сна (Journal)

#### 2.2.1 Создание записи

- **API**: `POST /api/journal/entries`
- **Входные данные**:
  ```json
  {
    "createdAt": "2025-01-07T08:00:00Z",
    "sleepHours": 7.5,
    "stressLevel": 4,
    "note": "Felt rested today"
  }
  ```
- **Дополнительные поля модели JournalEntry**:
  - `date` (LocalDate) - дата записи
  - `moodTag` (String) - тег настроения (HAPPY, NEUTRAL, SAD, STRESSED, RELAXED)
  - `caffeineIntake` (Integer) - потребление кофеина (чашки/мг)
  - `alcoholIntake` (Integer) - потребление алкоголя (порции)
  - `activityLevel` (String) - уровень активности (LOW, MEDIUM, HIGH)
  - `lastMealBeforeSleep` (Integer) - часов до сна с последнего приема пищи
  - `screenTimeBeforeSleep` (Integer) - минуты экранного времени перед сном

#### 2.2.2 Получение записей

- **API**: `GET /api/journal/entries`
- **Query параметры**:
  - `limit` (optional) - количество записей
  - `offset` (optional) - смещение для пагинации

#### 2.2.3 Операции с записью

- **GET /api/journal/entries/:id** - получение конкретной записи
- **DELETE /api/journal/entries/:id** - удаление записи

### 2.3 Модуль сессий сна (Sleep Sessions)

#### 2.3.1 Модель SleepSession

```java
@Entity
public class SleepSession {
    Long id;
    LocalDateTime startTime;      // Время начала сна
    LocalDateTime endTime;        // Время пробуждения
    User user;                    // Связь с пользователем
    String rawDataPath;           // Путь к сырым данным датчиков
    Integer qualityScore;         // Оценка качества сна (0-100)
}
```

#### 2.3.2 Функции

- Начало сессии отслеживания сна
- Завершение сессии с сохранением данных
- Получение истории сессий сна
- Расчет качества сна на основе данных

### 2.4 Модуль мониторинга стресса (Stress)

#### 2.4.1 Модель StressData

```java
@Entity
public class StressData {
    Long id;
    LocalDateTime timestamp;      // Время замера
    Double hrvScore;              // HRV (Heart Rate Variability)
    String stressLevel;           // LOW, MEDIUM, HIGH
    User user;
}
```

#### 2.4.2 API Endpoints

- **GET /api/analysis/stress** - анализ стресса за период
  - Query: `days` (default: 7)
  - Response: `averageStress`, `trend`, `insights[]`

## 7. Конфигурация

### 7.1 Backend (application.properties)

```properties
# Database
spring.datasource.url=jdbc:postgresql://localhost:5432/sleepdb
spring.datasource.username=postgres
spring.datasource.password=password
spring.jpa.hibernate.ddl-auto=update

# JWT
jwt.secret=your-256-bit-secret
jwt.expiration=3600000
jwt.refresh-expiration=604800000

# Server
server.port=8080
```

### 7.2 Mobile (.env)

```env
EXPO_PUBLIC_API_BASE_URL=http://localhost:8080
```

### 7.3 Docker Compose

```yaml
version: '3.8'
services:
  backend:
    build: ./sleep-backend
    ports:
      - "8080:8080"
    depends_on:
      - postgres
    environment:
      - SPRING_DATASOURCE_URL=jdbc:postgresql://postgres:5432/sleepdb

  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: sleepdb
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```
